    area workflows

    This page outlines workflows to solve the area problems with earth engine methods defined on the methods pages.

    Vector approach

    Here is a workflow to solve the vector-based approach to the area problem.

    Load modules.

    1. Load the data module.
    2. Load the tasks module.

    Define a SUBJECT.

    1. Load feature collection from address → data.cadastre.college.fc_address
    2. Print the number of objects in the collection. → Answer A below

    Calculate area of each SUBJECT feature.

    1. Area of features (acres) in FC.

    Print results.

    1. Print size of collection. → Answer B below
    2. Sum the values in a table column. → Answer C below
    Check your answers for vector solution.
    (A) Number of parcels owned by the college: 286
    (B) Number of objects in SUBJECT collection: 286
    (C) Total acres of college-owned parcels: 7009

    Theme within regions

    Work through the steps below to solve the theme within regions area problem.

    Load modules.

    1. Load the data module.
    2. Load the tasks module.

    Define CUTTERS.

    1. Load feature collection from address → data.cadastre.college.fc_address
    2. Print size collection. → Answer A below

    Define binary THEME.

    1. Load feature collection from address. → data.rarity.vt.nc.fc_address
    2. Inspect the properties of the feature collection.
      1. Print first feature in FC.
      2. Print unique values in FC.
        1. Explore ‘S_RANK’.
        2. Explore ‘S_NAME’.
      3. Convert FC to binary image. → THEME as input

    Define the DOUGH.

    1. Make pixel area image.
    2. Erase values at locations with binary. → pixel area and theme binary

    Compute Zonal Summary (zs).

    1. Zonal summary of dough within cutters.

    Print results.

    1. Print size of collection. → Answer B below
    2. Sum the values in a table column. → Answer C below
    Check your answers for THEME IN REGIONS raster solution.
    (A) Number of parcels owned by the college: 286
    (B) Number of objects in zs collection: 286
    (C) Total acres of rare natural communities on college-owned parcels: 1284

    Theme within sites across regions

    Work through the steps below to solve the theme within sites across regions area problem.

    Load modules.

    1. Load the data module.
    2. Load the tasks module.

    Define SITES.

    1. Load feature collection from address → data.cadastre.college.fc_address
    2. Print size of collection. → Answer A below
    3. Convert FC to binary image. → Use SITES as input.

    Define REGIONS.

    1. Load feature collection from address → data.gov.town.fc_address
    2. Filter collection by bounds. → towns that intersect college parcels
    3. Print the number of objects in the collection. → Answer ABbelow

    Define THEME.

    1. Load feature collection from address. → data.rarity.vt.nc.fc_address
    2. Inspect the properties of the feature collection.
      1. Print first feature in FC.
      2. Print unique values in FC.
        1. Explore ‘S_RANK’.
        2. Explore ‘S_NAME’.
    3. Convert FC to binary image. → THEME as input

    Define the DOUGH.

    1. Make pixel area image.
    2. Erase values at locations with binary. → pixel area and theme binary and sites binary

    Compute Zonal Summary (zs).

    1. Zonal summary of dough within cutters. → REGIONS as cutters!

    Print results.

    1. Print size of collection. → Answer C below
    2. Sum the values in a table column. → Answer D below
    Check your answers for THEME IN SITES ACROSS REGIONS raster solution.
    (A) Number of parcels owned by the college: 286
    (B) Number of towns that intersect college lands: 9
    (C) Number of REGIONS in the zs output: 9
    (D) Total acres of rare natural communities on college-owned parcels: 1301

    Challenge problem

    Here are some clues to solve the area challenge problem.

    • Your dough will need two bands.
      1. One of pixel areas for locations that are THEME and SITE.
      2. Another of pixel areas for locations that are THEME. 
    var dough = dough_theme_sites.addBands(dough_theme);
    • You will need to alter some parameters in the Chart to reflect that you are charting two series:
      1. yProperties
      2. .setSeriesNames
      3. colors
